Part I

The AI PDLC Foundation

Five mandatory layers every CMS team must implement — competitive advantage lives in structured context, not prompts.

1
Layer 1
Context Layer
The Real Moat

Competitive advantage is not prompts. It is context. No AI generation without structured context ingestion.

  • Architecture Decision Records (ADRs)
  • API contracts & domain rules
  • Compliance & security policies
  • Data schemas & code standards
  • Prompt & instruction catalog
Stored in GitHub, Trimble Backstage, and AI-accessible documentation layers.
2
Layer 2
Agent Role
Specialization

Move from generic AI to domain agents. Each CMS product defines specialized agents with scope boundaries.

  • Vista Domain Agent
  • Prism ERP Logic Agent
  • Financial Compliance Agent
  • Payroll Compliance Agent
  • Shared Infrastructure Agent
  • Integration Agent
Agents are parameterized, constrained execution systems — not magic.
3
Layer 3
Hybrid Human
+ Agent Teams

Developer role shifts from coder to conductor. You're designing orchestration patterns, not typing loops.

  • Human owns: Architecture, tradeoffs, final approval, risk
  • Agents own: Boilerplate, tests, docs, refactors, pattern enforcement, pre-review
4
Layer 4
Async Background
Execution

Introduce background coding agents. Reduced context switching, overnight feature generation, parallelization at scale.

  • Feature spec written
  • Agent branch spawned
  • Implementation + tests + docs generated
  • Pre-review agents validate security, architecture, performance
  • Human final review
5
Layer 5
Governance
& Audit Layer

Enterprise policy control is mandatory. No shadow AI.

  • Agent access controls & data sensitivity boundaries
  • Token budget controls
  • Audit trail of AI-generated code
  • SAST/DAST parity for AI output
  • Velocity, quality & adoption metrics
No shadow AI. Every agent action is traceable.
