Move from one-session pair programming to coordinated multi-agent pipelines. Claude shifts from responder to goal-seeking orchestrator with bounded tools, explicit gates, and auditable actions.

01

02
Enrich

03

Agent Architecture

Three Roles, One System

Every agentic pipeline in CMS is built from the same three role primitives — Orchestrator, Worker, Critic. Each has bounded scope and explicit rules.

🎯
Orchestrator
Goal Decomposer
Breaks goals into subtasks, delegates to Workers, aggregates results, never implements directly. The strategic layer of the agent mesh.
  • Delegates, never implements
  • Aggregates structured outputs
  • Escalates on low-confidence results
  • Maintains goal state across turns
⚙️
Worker
Domain Executor
Owns one service or repo boundary. Returns structured outputs only. Scoped to a single domain — payroll, compliance, integrations — and nothing else.
  • One service / one repo boundary
  • Structured outputs only
  • No direct commits to main
  • PR + review gate required
🔍
Critic
Policy Enforcer
Checks policy, contracts, and acceptance criteria before anything reaches human visibility. The last programmatic line of defense before a human gate.
  • Policy and contract validation
  • Acceptance criteria enforcement
  • Low confidence → escalate
  • No silent failures permitted
